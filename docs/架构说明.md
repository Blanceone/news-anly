# A股情报系统 — 架构说明

## 行业板块 vs 概念主题

```
                      ┌───────────────────┐
                      │    Tushare API     │
                      │ stock_basic.industry│
                      └───────┬───────────┘
                              │ 按行业字段分类, 110个行业
                              ▼
  ┌─────────────────────────────────────────────┐
  │           行业板块 (Industry Sectors)        │
  │  Tushare stock_basic + daily.pct_chg 聚合    │
  │  实时: 涨跌幅/上涨/下跌/成交额               │
  │  DB: sector_cache 表                        │
  └─────────────────────────────────────────────┘
```

```
                      ┌───────────────────┐
                      │  人工维护 10大主题   │
                      │ theme_stock_mapping │ ← 57只股票, benefit_level
                      └───────┬───────────┘
                              │
  ┌─────────────────────────────────────────────┐
  │           概念主题 (Concept Themes)           │
  │  theme_stock_mapping(10个)                  │
  │  + AKShare 概念板块(实时行情)                │
  │  + ThemeDiscovery 自动发现(玻璃基板等)        │
  │  + 静态兜底(AI芯片/光模块等)                  │
  │  无实时行情 → change=0                       │
  └─────────────────────────────────────────────┘
```

### 关键区别

| 维度 | 行业板块 | 概念主题 |
|------|---------|---------|
| 数据源 | Tushare `stock_basic.industry` 交易所分类 | 人工维护 `theme_stock_mapping` + AKShare 概念 |
| 数量 | 110 个 | 10~50 个 |
| 实时行情 | ✅ 有（涨跌幅/涨跌比/成交额） | AKShare 概念有，KG 主题无 |
| 股票归属 | Tushare `industry` 字段自动归类 | 人工设定 `theme_stock_mapping` |
| 代码位置 | `tui/db.py` `industry_sectors()` | `tui/db.py` `concept_themes()` |

## 知识图谱架构

```
   theme(主题)          CONTAINS          industry(行业)
       │                                       │
       │  CONTAINS                              │  CONTAINS
       ▼                                       ▼
 technology(技术)  ────  ENABLE  ────▶  product(产品)
       │                                       │
       │  USES (company→tech)                  │  BENEFIT (product→stock)
       │                                       │
       ▼                                       ▼
   company(公司)  ── SUPPLIER/CUSTOMER ──▶   stock(股票)
```

### 实体类型（5 层 + 公司）

| 层级 | 类型 | 示例 | 数量 |
|------|------|------|------|
| 主题 | theme | AI, 半导体, 机器人 | 10 |
| 行业 | industry | 集成电路, 新能源 | 11 |
| 技术 | technology | 先进封装, 机器视觉 | 9 |
| 产品 | product | GPU, 伺服电机 | 7 |
| 公司 | company | 华为, 英伟达, 台积电 | 10 |

### 关系类型

| 关系 | 含义 | 方向 | 跳数 |
|------|------|------|------|
| CONTAINS | 包含 | theme→industry→technology | 1~2 |
| ENABLE | 使能 | technology→product | 1 |
| BENEFIT | 利好 | product→stock | 1 |
| SUPPLIER | 供应商 | company→stock | 1 |
| CUSTOMER | 客户 | company→stock | 1 |
| USES | 使用 | company→technology/industry | 1 |

### BFS 推理引擎

```
knowledge_graph.reason(keywords, industry, companies)
    │
    ├── 1. LIKE 匹配 kg_entity.name / description
    │       keywords=["28nm","良率"] → entity="先进封装"(technology)
    │
    ├── 2. BFS 遍历边
    │       compound_weight = parent_weight * (edge_weight/100)
    │       深度 ≤ 4, 权重 ≥ 0.05 剪枝
    │       ⚠️ 跳过 COMPETITOR 边
    │
    └── 3. 合并 kg_direct_benefit（捷径）
            输出 [(stock_code, score, path_count)]
```

**当前规模**：47 实体 / 82 关系 / 42 直连受益

## 新闻 → 事件分析

```
新闻(title + content)
    │
    ▼
EventService.extract()
    │  LLM Prompt: 8种事件类型 + 关键词/行业/情绪/重要性
    │  3次重试, 温度=0.1
    ▼
结构化JSON:
{
  event_type: ORDER|EARNINGS|TECHNOLOGY|POLICY|MNA|CAPITAL|RISK|OTHER
  industry: "半导体"
  keywords: ["28nm", "良率突破"]
  sentiment: positive|negative|neutral
  importance: S|A|B|C
  novelty_score: 0-100
  entities: [{type, name}]
}
    │
    ▼
INSERT INTO event_analysis
UPDATE news (sentiment, category, impact)
    │
    ├── ▶ ThemeDiscovery.discover(keywords, industry)
    │       新关键词 → theme_candidate 表
    │       candidate(1) → observing(3+) → official(20+)
    │
    └── ▶ EventClustering.cluster_event(event_id, title, content)
           jieba 分词 + TF-IDF + cosine similarity > 0.3
           同一事件多篇报道 → event_cluster + event_cluster_map
```

### LLM 提取字段

| 字段 | 说明 | 必填 |
|------|------|------|
| event_type | ORDER / EARNINGS / TECHNOLOGY / POLICY / MNA / CAPITAL / RISK / OTHER | ✅ |
| event_subtype | 具体子类别 | |
| industry | 所属行业 | |
| sub_industry | 细分行业 | |
| sentiment | positive / negative / neutral | ✅ |
| importance | S(100) / A(80) / B(60) / C(40) | ✅ |
| novelty_score | 0~100 新颖度 | ✅ |
| keywords | 关键词列表 | |
| entities | 实体列表 [{type, name}] | |
| summary | 一句话总结 | |

### 事件分数计算

```python
event_score = importance_base * 0.7 + novelty_score * 0.3
# S=100, A=80, B=60, C=40
```

## 事件 → 股票关联

```
event_analysis {keywords, industry, companies}
    │
    ├──▶ StockService.process_event_stocks(event_id, event)
    │
    │       Phase 1: 关键词匹配（substring）
    │       │   all_text = industry + sub_industry + " ".join(keywords)
    │       │   for theme in INITIAL_THEMES:                   ← 10大主题
    │       │       if any(theme_kw in all_text):
    │       │           INSERT INTO event_stock_mapping          ← theme_stock_mapping
    │       │
    │       Phase 2: Embedding 语义匹配
    │           EmbeddingService.match_event(keywords, industry)
    │           TF-IDF char_wb ngram(2,4) + cosine similarity > 0.15
    │           匹配结果 → event_stock_mapping
    │
    ├──▶ KnowledgeGraph.reason(keywords, industry, companies)
    │       BFS 推理: entity → relation → stock
    │       INSERT INTO event_stock_mapping (match_reason="图谱推理(路径N条)")
    │
    ├──▶ MarketVerifier.verify_event(event_id, industry, keywords)
    │       Tushare 行业涨跌幅 + 涨跌比 + 成交额
    │       → market_confirmation.confirmation_score
    │       ⚠️ 仅交易时段运行（工作日 9:25~15:00）
    │
    └──▶ INSERT OR IGNORE INTO event_stock_mapping
           (event_id, stock_code, stock_name, benefit_level, benefit_score, match_reason)
```

### 股票关联来源对比

| 来源 | 匹配方式 | 特点 | 数据 |
|------|---------|------|------|
| StockService 关键词 | substring 匹配 10 主题 keywords | 精准、无泛化 | 57 只股票 |
| StockService Embedding | TF-IDF 余弦相似度 | 语义泛化、发现隐含关联 | 57 只股票 |
| KG BFS 推理 | 多跳关系推理 | 发现间接关联（供应商/客户等） | 42+ 只 |

## 最终评分 (V3)

```python
TotalScore = EventScore × 15%
           + BenefitScore(分层) × 20%
           + MarketScore × 15%
           + ThemeHeat(衰减) × 15%
           + ClusterHeat(生命周期) × 10%
           + LeaderScore × 15%
           + LifecycleScore × 10%
```

| 维度 | 权重 | 来源 | 说明 |
|------|------|------|------|
| EventScore | 15% | event_analysis | 重要性×0.7 + 新颖度×0.3 |
| BenefitScore | 20% | event_stock_mapping | 分层加权: DIRECT×1.0 / INDIRECT×0.8 / SENTIMENT×0.5 |
| MarketScore | 15% | market_confirmation | 行业涨跌幅+涨跌比+成交额 |
| ThemeHeat | 15% | theme_heat | 衰减后: raw_heat × exp(-days/3) |
| ClusterHeat | 10% | event_cluster | 生命周期加权: BIRTH(0.8)/GROWING(1.0)/PEAK(0.7)/DECLINING(0.3)/DEAD(0) |
| LeaderScore | 15% | stock_profile | 流动性×20%+活跃度×30%+题材数×20%+涨停历史×30% |
| LifecycleScore | 10% | event_cluster | 生命周期系数加权

## 完整数据流

```
采集 ──▶ AI分析 ──▶ 关键词匹配 ──▶ 图谱推理 ──▶ 市场验证 ──▶ 热度计算 ──▶ 综合评分
 │         │           │             │            │            │            │
 ▼         ▼           ▼             ▼            ▼            ▼            ▼
news.db  event_     event_stock_  event_stock_  market_     theme_heat  stock_score
        analysis    _mapping      _mapping      confirmation             recommendation
                                    ↑                                          ↑
                                kg_direct_benefit                          theme_heat
                                kg_relation                               event_cluster
```

### 数据库分布

**news.db**（3 表）
- `news` — 原始新闻
- `reports` — 汇总报告（保留，未使用）
- `event_analysis` — AI 事件分析结果

**stocks.db**（19 表）
- `theme_stock_mapping` — 主题→股票映射
- `event_stock_mapping` — 事件→股票映射 (含 benefit_type/benefit_path)
- `stock_score` — 综合评分
- `recommendation_result` — 推荐结果 (strategy_type: HOT/THEME/LATENT)
- `market_confirmation` — 市场验证
- `sector_cache` — 行业板块行情缓存
- `kg_entity` — 图谱实体
- `kg_relation` — 图谱关系
- `kg_direct_benefit` — 直连受益
- `theme_candidate` — 新发现主题候选
- `theme_embedding` — TF-IDF 向量
- `theme_heat` — 主题热度 (含 decay_heat/last_active_time)
- `event_cluster` — 事件簇 (含生命周期字段)
- `event_cluster_map` — 簇→事件映射
- `stock_basic` — 股票基础信息
- `stock_profile` — 股票画像+龙头评分
- `theme_limitup_stats` — 涨停热度
- `backtest_result` — 回测结果
- `backtest_trades` — 回测交易明细
