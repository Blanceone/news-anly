# 行业板块、主题图谱与知识图谱实现

## 概述

系统通过三层数据源 + 两套映射机制 + 图推理引擎，实现从"新闻事件"到"受益股票"的全链路关联。整体架构：

```
Tushare (110行业)          KG实体 (37个)           AKShare (概念板块)
     │                          │                        │
     ▼                          ▼                        ▼
  ┌─────────────────────────────────────────────────────────┐
  │              all_sectors() 三层合并 → 129板块            │
  │  去重: name 相同则不重复添加                              │
  └──────────────┬──────────────────────────────────────────┘
                 │
    ┌────────────┴────────────┐
    ▼                         ▼
theme_stock_mapping      event_stock_mapping
(57只股票, 10主题)        (事件→股票, 实时匹配)
    │                         │
    └────────────┬────────────┘
                 ▼
          scoring_engine
      事件×30% + 受益×40% + 市场×30%
```

## 一、数据来源与三板聚合

### 1.1 Tushare 主力 — `_ts_sectors()`

**位置**: `tui/db.py:166`

- 调用 `pro.stock_basic(fields='ts_code,industry')` 获取全市场 ~5500 只股票的行业分类
- 获取最近一个交易日行情 `pro.daily(trade_date=…)` -> `pct_chg`, `amount`
- 按 industry 字段聚合：平均涨跌幅、上涨/下跌家数、成交总额
- 结果写入 `sector_cache` 表（`stocks.db`）作为离线缓存

```python
# 聚合逻辑示意
for ind, codes in industry_map.items():
    changes = [chg_map[c]["pct_chg"] for c in codes if c in chg_map]
    avg = sum(changes) / len(changes)
    up = count(c for c in changes if c > 0)
    down = count(c for c in changes if c < 0)
    total_vol = sum(chg_map[c]["amount"] for c in codes)
```

### 1.2 KG 主题 — `_kg_themes()`

**位置**: `tui/db.py:99`

四种来源合并去重：

| 来源 | 表 | 字段 |
|------|-----|------|
| 主题映射 | `theme_stock_mapping` | `theme_name` |
| 知识图谱实体 | `kg_entity` | `name WHERE entity_type='theme'` |
| 事件行业 | `event_analysis` | `industry`（近72小时） |
| 静态兜底 | 硬编码 20 个主题 | 确保非交易时段仍有数据 |

### 1.3 AKShare 概念补充 — `_ak_concept_sectors()`

**位置**: `tui/db.py:130`

- 调用 `ak.stock_board_concept_spot_em()` 获取东方财富概念板块实时行情
- 作为 Tushare 行业的补充（Tushare 只有标准行业，缺少概念如"机器人"、"AI"）

### 1.4 合并逻辑 — `all_sectors()`

**位置**: `tui/db.py:73`

```
1. Tushare 行业 (110个) — 主力，包含涨跌幅/涨跌比/成交额
2. KG 主题 — 补充行业未覆盖的主题，change=0
```

### 2.2 动态映射 — `event_stock_mapping`

**位置**: `services/stock_service.py:198`

事件到来时的实时匹配流程：

```
AI事件分析输出
  ├─ keywords: ["AI", "大模型", ...]
  ├─ industry: "人工智能"
  └─ sub_industry: "AI芯片"
         │
         ▼
  match_event_to_stocks(event_dict)
         │
         ├─ 将 industry + sub_industry + keywords 拼接为 all_text
         ├─ 遍历 INITIAL_THEMES 的 keywords
         │   └─ 若有任何 keyword 命中 all_text → 该主题命中
         └─ 从 theme_stock_mapping 取出该主题下所有股票
```

结果写入 `event_stock_mapping(event_id, stock_code, benefit_level, benefit_score, match_reason)`。

### 2.3 双重映射写入点

- **StockService.process_event_stocks()** — 关键词匹配

- 两者都写 `event_stock_mapping`，用 `INSERT`（非 `IGNORE`）允许重复
- 评分引擎用 `DISTINCT` 去重

## 三、知识图谱推理

### 3.1 图谱结构

**定义**: `services/knowledge_graph.py:13` `SEED`

```
Theme → Industry → Technology → Product → Stock
```

5 种实体类型，4 种关系：

| 层级 | 实体数 | 示例 |
|------|--------|------|
| Theme | 10 | 人工智能、算力基础设施、半导体… |
| Industry | 8 | AI芯片、光模块、服务器… |
| Technology | 7 | Chiplet、HBM、CPO… |
| Product | 5 | GPU芯片、AI服务器产品… |
| Stock | (直连42只) | 通过 `direct_benefit` 表 |

| 关系 | 语义 | 示例 |
|------|------|------|
| CONTAINS | 包含 | Theme → Industry, Industry → Tech |
| ENABLE | 使能 | Technology → Product |
| BENEFIT | 受益 | Product → Stock |
| (direct_benefit) | 直连受益 | Theme → Stock (跳过中间跳) |

### 3.2 BFS 推理引擎

**入口**: `KnowledgeGraph.reason(keywords, industry)`

```
事件关键词 + 行业
       │
       ▼
  search_entities(keywords, industry)
   └─ LIKE 匹配实体 name 和 description
       │
       ▼
  _bfs(entity_type, entity_name, compound_weight=1.0, depth=0)
   │
   ├─ 终止条件: depth > 4 || compound_weight < 0.05
   ├─ 已访问剪枝: visited set
   │
   ├─ 若当前为 stock:
   │    └─ 记录 max_weight, 记录路径
   │
   ├─ 从 kg_relation 查出子节点:
   │    └─ compound_weight = parent_weight × (edge_weight / 100)
   │
   └─ 从 kg_direct_benefit 查出直连股票:
        └─ compound_weight = parent_weight × (weight / 100)
```

**权重计算示例**:

```
事件命中 "AI芯片"(entity)
  └─ CONTAINS "Chiplet"(tech, w=85)
       └─ ENABLE "封装服务"(product, w=95)
            └─ BENEFIT "长电科技"(stock, w=95)
                 └─ compound = 1.0 × 0.85 × 0.95 × 0.95 = 76.7 → score = 77
```

最终按 `score × 100` 降序，取 Top 20。

### 3.3 直连受益表 `kg_direct_benefit`

42 条记录，直接从实体层级跳转到股票，跳过中间层级。用于 BFS 中快速命中股票。

示例：
```
人工智能 → 科大讯飞(w=95): AI语音龙头
算力基础设施 → 工业富联(w=95): AI服务器龙头
半导体 → 中芯国际(w=95): 晶圆代工龙头
```

## 四、市场验证与板块行情

### 4.1 事件→板块映射

**位置**: `services/market_verifier.py:12` `INDUSTRY_BOARD`

将事件识别出的 `industry` 映射到 A 股行业板块名称：

```
"AI芯片"       → "半导体"
"光模块"       → "通信"
"人工智能"     → "计算机应用"
"低空经济"     → "国防军工"
```

未精确匹配时，使用 `BOARD_KEYWORDS` 做关键词模糊匹配。

### 4.2 板块行情评分

**位置**: `market_verifier.py:210` `_compute_score()`

```
涨跌幅得分(0-40) + 涨跌比得分(0-30) + 成交额得分(0-30) = MarketScore(0-100)
```

| 条件 | 分值 |
|------|------|
| 涨幅≥5% | 40分 |
| 涨跌比≥80% | 30分 |
| 成交额≥500亿 | 30分 |
| 非交易时段 | 0分（直接返回） |

### 4.3 评分引擎集成

**位置**: `services/scoring_engine.py:109`

```
TotalScore = EventScore × 30% + BenefitScore × 40% + MarketScore × 30%
```

- **EventScore**: 取该股票关联事件中最大事件评分
- **BenefitScore**: 取该股票关联事件中最大受益评分（level 1→95, 2→80, 3→60）
- **MarketScore**: 对应板块的市场验证评分

## 六、TUI 展示

### ThemeViewScreen (`tui/screens/theme_view.py`)

左右两栏布局：

- **左栏**: 129 个板块（Tushare + KG + 概念合并），按涨跌幅排序，显示涨跌幅/上涨家数/下跌家数/成交额
- **右栏**: 选中板块的成分股，按涨跌幅排序

数据流程：

```
on_mount → all_sectors() → left panel
用户点击 → sector_stocks(sector_name) → right panel
    └─ 四层兜底:
       1. Tushare industry 过滤 (stock_basic.industry)
       2. KG 主题映射 (theme_stock_mapping)
       3. AKShare 行业成分 (stock_board_industry_cons_em)
       4. AKShare 概念成分 (stock_board_concept_cons_em)
```

### StockViewScreen (`tui/screens/stock_view.py`)

点击股票查看其关联事件，事件时间自动从 UTC 转为北京时间（`to_bjt()`）。

## 七、数据流完整链路

```
新闻采集 → AI事件识别 → StockService.match() → event_stock_mapping
                                            → KnowledgeGraph.reason() → event_stock_mapping
                                            → MarketVerifier.verify() → market_confirmation
                                                                       ↓
                                                               ScoringEngine.calculate()
                                                                       ↓
                                                               stock_score + recommendation_result
                                                                       ↓
                                                                TUI 各屏幕消费
```
